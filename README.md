#### Point
<p>I don't share how to install git or creating remote account in Github, Gitlab or other remote websites.
But it's crucial to choose your name (id) and your email address for Git installation properly because Git uses it as encryption keys to assign your work. However it can be changed after installation but not recommended</p>

<p>You can check what id and gmail set for your git after installing with commands.</p>
<p>First open Git Bash, write commands below</p>

```
git config --global user.name
git config --global user.email
```
<p>It will show your global configuration which is set for Git.</p>

### .gitignore
<p>In real projects for managing security, we don't want to address some of our assets, keys, passwords and other type of secret files.</p>
<p>On one hand, we can move our secret things out of the project folder and use absolute or relative path to use them. On the other hand, with changing of some modules' built-in properties as an essential accessible core, manner and workflow of our connections come into a serious problem.</p>
<p>In that case, we use <b><i>.gitignore</i></b> file to tell git not to track the changes or content of it.</p>
